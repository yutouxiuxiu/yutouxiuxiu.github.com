零、什么是用例图

1.用例（Use Case）

帮助角色确定系统使用情况的UML组件

2.用例组（Use Case Group）

从用户角度出发对如何使用系统的描述

3.用例图（Use Case Diagram）

用图形的方式来描述情景 由用例/用例组构成

作用

描述需求场景

与用户交流过程中 快速建立用户对业务的描述 并且和用户确认 不仅要记录客户告诉我们要什么 还要发觉他们没有告诉我们的

设计好的用例图用来检验程序实现场景

 

一、用例的要素

参与者（主参与者、附加参与者、后台参与者）

假设条件

前置条件

场景中的步骤

后置条件

 

二、用例图元素

1.包 Package

（文件夹）

分类 模块 子模块

2.用例 Use Case

（椭圆）

实际的具体的动作（用动宾短语）

 

3.角色 Action

（小人）

执行者（主参与者、附加参与者、后台参与者）

属性：

name  -  名称

Stereotype  -  立体类型 它是一个怎样的用户 是一个什么类别的角色 会以<<  >>来表示

Visibility  -  因为角色也是一个类 所以它也有 PUBLIC PRIVATE等

IsAbstract  -  是否是抽象

Attributes  -  属性

可以添加属性 属性的属性如下：

Name  -  属性名

Stereotype

Type  -  属性的类型 Interge/ Float/ Boolean/ String

InitialValue  -  初始值

4.双向关联 Association

（实线）

 

5.定向关联 DirectedAssociation

（带箭头实线）

陈述句 把角色和用例连接起来 比如用户是芋头 用例是喝水 关联就是芋头喝水

如果是连接两个用例 就是然后的意思

属性：

General(End?)

End?.Multiplict  -  对应关系（这是类图的）

一对多  -  一边是1 一边是*

6.泛化 Generalization

（实线加空心箭头）

就是继承

7.依赖 Dependency

（虚线加箭头）

把两个用例连接起来 比如喝水依赖拿杯子

8.包含 Include

（虚线箭头加<<include>>）

特殊的依赖

9.扩展 Extend

（虚线箭头加<<extend>>）

特殊的依赖

10.系统边界 System Boundary

（矩形加system）

将多个用例划分到一个小的范围内

 

另外 在Annotation里面 一个Note和虚线连接到某个元素上 就是注释

 

在StarUML中 要想删除元素 ctrl+DEL是彻底删除

 

三、栗子

1.用户登录

并且校验信息中包含Username和Password属性

System系统边界 就是指 这部分是系统做的事情

 

还可以使用大图 就是不同类型的图拥有的元素放在一张图中

2.超市购物

如果两条线 重叠了 在中间拉出一个点 就分开了 然后可以按ctrl + l将线自动变直

除了主执行者之外的执行者 都不用关联去连接 而是使用依赖 这样便于找到用例图的入口
