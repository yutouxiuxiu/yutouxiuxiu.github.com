一、Spring MVC

1.Spring MVC都做了什么

 

2.Spring MVC的核心组件

（1）DespatcherServlet（系统提供）

请求都要通过这个进行转发 转发到具体的Controller 就像Struts的filter

（2）Controller

控制器

（3）HandlerMapping（系统提供）

映射处理器 当一个请求过来的时候从DespatcherServlet转发到Controller 这个转发的策略 就是HandlerMapping

（4）ModelAndView

Model是获取到的数据 View是页面 这个模块将数据和页面结合起来 将数据和页面封装起来

（5）ViewResolver（系统提供）

视图解析器 把ModelAndView解析成具体的jsp页面 给展示出来

（6）Interceptors

拦截器

三、三种HandlerMapping映射处理器

1.BeanNameHandlerMapping（不配置默认是这个）

<bean class="org.springframework.web.servlet.handler.BeanNameHandlerMapping"></

2.简单URL

手动配置映射关系

<bean class="org.springframework.web.servlet.handler.SimpleUrlHandlerMapping">

<property name="mappings">

<props>

<prop key="/hi.do">Controller的bean的id</

......

</

</

</

（注意！！！）

如果同时定义BeanNameHandlerMapping和简单URL 在Controller的bean中进行配置的name依然可以访问

映射处理器之间独立 互不影响

但是如果只定义简单URL bean中的name就不起作用了

如果什么都没定义 才默认有BeanNameHandlerMapping

3.ControllerClassNameHandlerMapping

<bean class="org.springframework.web.servlet.handler.ControllerClassNameHandlerMapping"></

然后访问 controller的类名.do 进行访问 就可以访问到

注意！！！访问时 类名使用首字母小写访问

想要访问这个Controller当然有三种方式了

可以在简单URL映射处理器中配置一个 （需要指定id属性）

或者使用bean的name

...

这样一个ParameterizableViewController就对应一个jsp页面 要对应多个jsp页面 就配置多个ParameterizableViewController的bean即可

2.AbstractCommandController命令控制器（过时）

方法参数中的Object obj用来接收前台传来的数据

它接收一个实体类（Model） 不知道是什么类型 需要进行配置

<bean id="commController" class="com.yutouxiuxiu.controller.CommController">

<!--指定接收数据的数据类型-->

<property name="commandClass" value="com.yutouxiuxiu.model.Person"></

</

比如说实体类Person中有id、name、age等属性 访问如下的地址

comm.do?id=1&name=liuneng&age=22

访问一个带有对应参数的地址的时候 obj参数就会按照配置中指定的实体类的set方法去匹配url参数 通过反射的方式 注入到obj里面

这个url参数的参数名首字母可以大写也可以小写

3.SimpleFormController表单控制器

form.jsp

<bean id="formController" name="/form.do" class="com.yutouxiuxiu.controller.FormController">

所以说 表单页面的method如果指定成GET就会跳回到表单页面 无法提交

五、注解方式进行配置

当然 使用注解的方式进行配置的时候也需要在web.xml中定义这个DispatcherServlet

如果需要 ViewResolver和Interceptors也是在配置文件中配置的

而Controller和Mapping就在注解中配置了

不过还是需要在配置文件中配置一下注释驱动和扫描器

<!--注解驱动-->

<mvc:annotation-driven/>

<!--扫描器 扫描Controller-->

<context:component-scan base-package="com.yutouxiuxiu.controller"/>

这样Controller就不用继承和实现了

2.封装数据到页面中

可以把页面和数据封装在一起 交由视图解析器进行解析

这就类似于smarty 把页面和assign的数据封装到一个对象里 然后交由模板解析器解析

@RequestMapping("/test.do")

public ModelAndView test() {

Map<String, Object> map = new HashMap<String, Object>();

Person person = new Person();

person.setId(1);

person.setName("yutou");

map.put("p", person);  //相当于request.setAttribute("p", person); map里的数据 就是结果集里的数据

return new ModelAndView("index", map);

}

然后就可以在index.jsp的页面中使用EL表达式进行值的获取

${p.id}

${p.name}

上面的写法比较麻烦 可以按照下面的方式来简化书写

@RequestMapping("/test.do")

public String test(Map<String, Object> map) {  //在参数列表里面定义map

map.put(...);

return "index";

}

通常使用这种方式！！！

@RequestMapping("/test.do")

public String test(Model model) { 

mode.addAttribute(p, "person");  //更加像smarty中的assign了

return "index";

}

 

3.指定请求方式

@RequestMapping(value="/test.do", method=RequestMethod.GET)

如果指定了请求方式 使用其他的请求方式就不能访问到了 会报405错误 提高了安全性

5.重定向

地址栏会发生变化

（1）Controller内部重定向

@RequestMapping("/redirect.do")

public String redirect() {

return "redirect:hi.do";  //"redirect:"加上其他方法的RequestMapping路径

}

（2）重定向到其他Controller的方法

@RequestMapping("/redirect.do")

public String redirect() {

return "redirect:/Controller的RequestMapping路径/hi.do";  //（注意！！！）最前面必须要加上"/"表示退回根目录中 要是不加 就还是在当前的Controller的RequestMapping基础上加上redirect路径

}

 

六、文件上传

配置

<bean id="必须是multipartResolver" class="org.springframework.web.multipart.commons.CommonMultipartResolver">

<property name="maxUploadSize" value="以Byte为单位"></

</

表单

<form enctype="multipart/form-data">

<input type="file" name="file" />

</

Controller接收

@RequestMapping("upload.do")

public String upload(Person person, HttpServletRequest request) {

//1.转换request为复杂类型的

MultipartHttpServletRequest mRequest = (MultipartHttpServletRequest) request;

//2.拿到文件

CommonMultipartFile cFile = (CommonMultipartFile) request.getFile("file");  //根据表单项的name获取

//3.获得文件的字节数组

byte[] fByte = cFile.getBytes();

//4.获取根目录

String realPath = request.getSession().getServletContext().getRealPath("");

//5.输出到文件

OutputStream out = FileOutputStream(new File(realPath + "/upload/" + cFile.getOriginalFilename()));

out.write(fByte);

out.flush();

out.close();

return null;

}

 

七、Interceptors拦截器

1.三种拦截器

前置、后置、最终

 

（1）preHandle前置拦截

Object参数是拦截的Controller类

在Controller执行之前执行

如果返回true 执行Controller 如果返回false 不执行

最常用在权限拦截

（2）postHandler后置拦截

ModelAndView参数

在Controller执行完毕 视图解析器执行之前执行

目的是对ModelAndView进行统一的修改 比如

Map<String, Object> map = modelAndView.getModel();

map.put("test", "append something");

（3）afterCmpletion最终拦截

Exception参数

在解析器解析完成之后执行

目的是将出现的异常进行记录

 

2.配置拦截器

<mvc:interceptors>

<mvn:interceptor>

<bean class="com.yutouxiuxiu.interceptors.MyInterceptor"></

</
