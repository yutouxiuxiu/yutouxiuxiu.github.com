2.日志技术

（1）jdk logging

在jdk的开发包内 不用导入jar包

java.util.logging

（2）log4j

apache公司的开源框架

（3）common-loggings

apache的 通用的日志 用户可以自由选择第三方的日志组件做实现

（4）slf4j

simple logging facade for java 简单的门面 不是具体的日志解决方案 它能把所有的日志统一

前两种算是两种不同的日志实现

后面两种主要是在做一些日志接口的统一工作

spring用的common-loggins hibernate用的是slf4j 但是他们用的日志的实现都是log4j

 

一、log4j

log4j 2 我也没用过 现在一般都用log4j 1

1.导入jar包

2.配置文件

log4j.xml

log4j.properties（推荐）

（1）log4j有三个部件

记录器  -  用于设置日志的输出级别和日志的输出源 一个记录器可以对应多个输出源

记录器 = 级别, 输出源1, ...

log4j.rootLogger  -  根记录器 所有日志的默认配置

日志级别

fatal  -  致命错误

error  -  普通错误（一般在catch里记录）

warn  -  警告信息

info  -  普通信息

debug  -  调试信息

trace  -  堆栈信息

中间的四个比较常用

log4j.rootLogger = DEBUG, stdout, file

当记录日志的时候 只会向设置的输出源输出比设置级别高或等于的信息

设置记录级别的时候 有8个选项 除了这6个级别 还有两个选项

off - 全关

all - 全开

常用级别

error（项目上线后）、warn、info、debug（开发时）

输出源  -  日志输出的目的地

log4j.appender.输出源名字 = 输出源实现类

log4j.appender.输出源名字 = org.apache.log4j.ConsoleAppender

org.apache.log4j.ConsoleAppender  -  控制台

org.apache.log4j.FileAppender  -  文件

org.apache.log4j.DailyRollingFileAppender  -  每天文件

org.apache.log4j.RollingFile  -  到达指定大小产生新文件

是控制台时

log4j.appender.输出源名字.Target = System.err

System.out  -  黑色的

System.err  -  红色的

是文件时

log4j.appender.输出源名字.File = d:/...

 

也可以记录到数据库/邮箱中

布局

log4j.appender.输出源名字.layout = 布局实现类

PatternLayout  -  自定义布局 灵活性高

如果用PatternLayout

log4j.appender.输出源名字.layou.ConversionPattern = 设置格式

$d{ABSOLUTE} %5p %c{1}: %L - %m%n

日期 级别 类: 行号 - 信息 回车

 

3.记录日志

private static final Logger LOGGER = Logger.getLogger(当前类.class);

...

//记录日志

LOGGER.fatal("致命错误");

LOGGER.error("普通错误");

LOGGER.warn("警告信息");

LOGGER.info("普通信息");

LOGGER.debug("调试信息");

LOGGER.trace("堆栈信息");

 

比如设置的级别是debug 堆栈信息会不显示

 

4.不同包的日志记录器记录不同级别

log4j.rootLogger = off, stdout, file

log4j.logger.包名 = 日志级别

更便于我们观察某个包中的内容 易于分析
