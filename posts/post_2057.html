php面向对象 - 类的自动加载

{ post:time }
一、为什么要自动加载

在实际的网站开发中 有很多类 都放在不同的文件中 比如说 UserModel GoodsModel CartModel...

要使用的时候 就要include 还不知道之前是否已经include过

这时就可以使用自动加载

 

二、如何使用

1.__autoload

如果调用某个不存在的类 在报错之前 还有一次介入的机会

这时 系统会自动调用__autoload方法 并传递类名

function __autoload($class) {
    echo '类',$class,'就要加载了';
}

$userModel = new UserModel();

/*
类UserModel就要加载了
( ! ) Fatal error: Class 'UserModel' not found in D:\wamp\www\study\autoload.php on line 7
*/

一般不要这么写 而采用注册方式 避免多自动加载函数的冲突问题

2.自动加载

function __autoload($class) {
    require('./' . $class . '.class.php');  //这里一定要精细判断要加载的类 指定哪些类在这里自动加载 这样才比较不容易和第三方类库中的自动加载冲突
}

类定义是全局有效的 不管在不在函数中定义

 

（注意！！！）使用第三方库的时候 它很有可能也使用了自动加载

所以可能会和我们自己的自动加载冲突 比如说smarty3的自动加载

php为了能让多方各自有自己的自动加载函数

提供了自动加载注册机制来解决这个问题 可以注册多个 会全都按照先后顺序执行

spl_sutoload_register('自动加载函数名');

不过这样可能会报warning 因为使用第一个自动加载函数没加载到 会报警告 而第二个自动加载函数加载到了

使用这个函数注册之后 将不会再执行__autoload函数

3.ThinkPHP中的应用

spl_autoload_register('autoload');  //注册自定义方法作为自动加载函数
