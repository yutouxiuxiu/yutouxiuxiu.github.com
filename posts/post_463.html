JSTL

{ post:time }
一、简述

JSTL（JavaServerPages Standard Tag Library）JSP的标准标签库

和EL表达式结合 可以完全替代<%%> 提高程序的可读性和可维护性

EL表达式 不能做判断和循环 JSTL可以完成这个功能

版本：

1.0 - JavaEE1.3 1.4之前EL还没有被纳入正式的官方规范

1.1 - JavaEE1.4

1.2 - 和1.1几乎差不多 没什么太大变化

二、使用

1.需要下载引入支持jar包

jstl.jar  -  使用到的接口类

standard.jar  -  使用到的具体实现类

   |--  META-INF

      |— *.tld  -  所有的标签库

JavaEE5 之后 都内置了jstl包 不用导了

2.在jsp文件中引入标签

引入c标签库（c.tld）

看它的文件里面有 short-name和uri 把他们用taglib引入

<%@ taglib uri=”http://java.sun.com/jsp/jstl/core” prefix=”c” %>

（注意！！！） 如果uri写错了 比如少写了一个jsp 它就变成了1.0版本的 就不支持EL表达式了

http://java.sun.com/jsp/jstl/core  -  1.1

http://java.sun.com/jstl/core – 1.0

http://java.sun.com/jstl/core-rt – 1.0 并支持EL

3.使用标签

<c:…/>

 

举个栗子

<c: set var=”n” value=”10” scope=”page”></c: set>

<c:if test=”${ n > 5 }”>

   n > 5

</c:if>

<c:if test=”${ n <= 5 }”>

   n <= 5

</c:if>

 

三、核心标签库（Core）

还有SQL标签，XML标签，国际化用的标签 不过暂时用不到

1.out

将内容输出到页面

属性：

value  -  输出的内容 常量 或 变量的EL表达式

default  -  默认值

escapeXml  -  默认为true 是否转义HTML

（1）输出常量

<c: out value=”hello yutou”></c: out>

（2）输出变量

    先用set标签设置属性

<c: out value=”${ 变量名 }”></c: out>

（3）输出默认值

    如果没有设置city的值 就输出默认值

<c: out value=”${ city }” default=”北京”></c: out>

（4）转义HTML代码输出

直接显示HTML的内容

<c: out value=”<a href=’#’>link</a>”>

会输出<a href=’#’>link</a> 而不会变成一个链接 要输出成一个链接 请参考escapeXml

2.set

属性：

var  -  变量名称

value  -  变量值

scope  -  作用范围（page, request, session, application）

target  -  作用目标对象

property  -  对象的属性名称

（1）设置值到四个作用范围

<c: set var=”name” value=”value” scope=”page”></c: set>

（2）设置属性值到四个范围中的JavaBean对象

<%

Person p = new Person();

p.setName(“yutou”);

pageContext.setAttribute(“p”, p);

%>

<c: set target=”${ p }” property=”name” value=”xiuxiu”></c: set>

3.remove

属性：

var  -  变量名

scope  -  作用范围 默认是page

移除属性

<%

pageContext.setAttribute(“name”, “yutou”);

%>

<c:remove var=”name” scope=”page” />

4.catch

捕获标签内出现的异常 封装到var所指定的对象里

<c:catch var=”e”>

  <%

    int a = 10 / 0;

  %>

</c:catch>

${ e }

${ e.message }

5.if

条件判断标签 注意 没有else

属性：

test  -  必须要写的属性 判断的条件 里面写EL表达式

var  -  把test的执行结果存在某个变量里

scope  -  var变量的作用范围

6.choose

多重选择

<c:choose>

  <c:when test=”${ weekdy == 1 }”>

    星期一

  </c:when>

  <c:when test=”${ weekdy == 2 }”>

    星期二

  </c:when>

  <c: otherwise>

    休息

  </c: otherwise>

</c:choose>

7.forEach

for循环

属性：

var  -  遍历过程值

items  -  要被遍历的数组

begin  -  开始

end  -  结束

step  -  步长 默认为1

varStatus  -  用于记录保存迭代的信息

varStatus

index - 当前成员的索引

count - 当前循环次数

first - 当前是否为第一个成员

last - 当前是否为最后一个成员

（1）遍历数组，List集合

<%

  pageContext.setAttribute(“arr”, new int[] = {11, 22, 33, 44, 55, 66});

%>

<c:forEach var=”i” items=”${ arr }”>

  ${ i }

</c:forEach>

（2）遍历Map集合

<%

  Map map = new HashMap():

  map.put(“name1”, “yutou”);

  map.put(“name2”, “xiuxiu”);

  pageContext.setAttribute(“map”, map);

%>

<c:forEach var=”entry” items=”${ map }”>

  ${ entry.key } = ${ entry.value }

</c:forEach>

（3）1~10

<c:forEach var=”i” begin=”1” end=”10” >

  ${ i }

</c:forEach>

（4）1~100中的基数 每第三个数变红

<c:forEach var=”i” begin=”1” end=”100” step=”2” varStatus=”status”>

  <c:if test=”${ status.count % 3 == 0 }”>

    <font color=”red”>${ i }</font>

  </c:if>

  <c:if test=”${ status.count % 3 != 0 }”>

    ${ i }

  </c:if>

</c:forEach>

8.forTokens

切分字符串

属性：

var  -  切分后循环每次切分出的结果放在变量里

items  -  要切分的数据

delims  -  按照什么切分

传统方式切分出get请求的参数

String[] s = url.split(“\\?|&”);

forTokens方法

<c:forTokens var=”i” items=”yuotu,xiuxiu,yuyuan” delims=”,”><c:forTokens>

9.import

导入

属性：

var  -  把页面的代码存到的变量中

url  -  引入页面的url（会在context下的url查找文件）

<context  -  引入页面的工程名

导入页面

<c:import var=”i” url=”” context=””>

  ${ i }

</c:import>

10.url

url重写 禁用cookie时session追踪时用的 

传统方式

String url = “”;

url = response.encodeURL(url);

标签方式

<c:url context=”项目名称” value=”页面”>

  <c: param name=”” value=””></c: param>  还可以用get方式传参数

</c:url>

11.redirect

重定向

<c:redirect context=”项目名称” value=”页面”></c:redirect>
