零、引例

一级缓存、二级缓存是对象缓存 缓存的是表中所有的数据

如果只要查很多个字段中的某几个字段 但是一用get 就把所有的值都查过来了 大大的浪费内存资源

引出了查询缓存

一、查询缓存

1.查询缓存是数据缓存 缓存的是数据 不是对象

2.查询缓存是建立在二级缓存基础之上的 只有这个类建立了二级缓存 才能有查询缓存

3.查询缓存只针对Query对象有效

 

  • 查询缓存使用步骤

1.开启二级缓存

2.指定某个类开启二级缓存

3.开启查询缓存

<property name="cache.use_query_cache">true</property>

4.使用查询缓存

Session session = sessionFactory.getCurrentSession();

Transaction transaction = session.beginTransaction();

Query query = session.createQuery("from Student");

query.setCacheable(true);    //因为查询缓存是封装在Query中的 意思是把数据可以缓存到查询缓存中

List<Student> students = query.list();    //把数据（不是对象）放入到查询缓存中

transaction.commit();

5.验证

session = sessionFactory.getCurrentSession();

transaction = session.beginTransaction();

query = session.createQuery("from Student");

List<Student> students = query.list();

transaction.commit();

发现第二次查 学生表就不查了

数据缓存是根据查询语句来标识的 HQuery语句一致 就利用缓存 不一致就不利用

query.list

1.在一些情况下 可以操作一级缓存 二级缓存

2.但也可以不操作一级缓存 二级缓存（插入操作）

3.在进行查询的时候利用的是查询缓存 而不是一级缓存 二级缓存

 

查询缓存应用的场景

 

查询的字段数远小于数据库中的字段 会按照需求缓存数据 通过查询语句查询
