零、什么是线程间通信

线程间通信 其实就是多个线程在操作同一个资源 但是操作的动作不同 也就是说执行的不是一个run函数

 

一、线程间通信的示例

class Resource {

string name;

string sex;

}

class Input implements Runnable {

private Resource r;

Input(Resource r) {

this.r = r;

}

public void run() {

int x = 0;

while (true) {

//synchronized(r) {

if (x%2 == 0) {

r.name = 'yutou';

r.sex = 'girl';

} else {

r.name = '修修';

r.sex = '男';

} 

x++;

//}

}

}

}

private Resource r;

Output(Resource r) {

this.r = r;

}

 

public void run(r) {

while (true) {

//synchronized() {

//}

}

}

这样会出现奇奇怪怪的问题 就是

yutou 男

修修 girl

因为 可能刚把name改成修修的时候 sex还是原来的girl呢 output就得到了执行权 然后进行了输出 就出现了上面的问题

解决办法

加同步锁 把上面代码中被注释掉的synchronized恢复即可

 

二、等待唤醒机制

两个线程操作共享数据 一个做完了 等待 另一个再做 唤醒第一个 然后等待

等待的线程 都在内存中的线程池里 唤醒的时候 唤醒的是第一个

如果想全部唤醒 就用notifyAll

注意 wait notify notifyAll 都是Object的方法 并且当前线程必须拥有此对象监视器 换句话说 就是必须在同步代码块中才能行

因为这些方法在操作同步中线程时 都必须要标识他们所操作线程持有的锁

只有同一个锁上的被等待的线程 可以被同一个锁上的notify唤醒

 

等待和唤醒必须是同一把锁

这就是为什么这三个方法上是定义在Object类上的了 因为锁是任意的

class Resource {

string name;

string sex;

boolean flag = false;

}

class Input implements Runnable {

private Resource r;

Input(Resource r) {

this.r = r;

}

public void run() {

int x = 0;

while (true) {

synchronized(r) {

if (r.flag) try { r.wait(); } catch (Exception e) {}

if (x%2 == 0) {

r.name = 'yutou';

r.sex = 'girl';

} else {

r.name = '修修';

r.sex = '男';

} 

x++;

r.flag = true;

r.notify();

}

}

}

}

class Output implements Runnable {

 

private Resource r;

Output(Resource r) {

this.r = r;

}

 

public void run(r) {

while (true) {

synchronized() {

if (!r.flag) try { r.wait(); } catch (Exception e) {}

syso(r.name + " " + r.sex);

r.flag = false;

r.notify();

 

}

}

}

}

Thread t2 = new Thread(out);

class Resource {

private String name;

private String sex;

private boolean flag = false;

public synchronized void set (String name, String sex) {

if (flag) try { this.wait(); } catch (Exception e) {}  //同步函数的锁是this

this.name = name;

this.sex = sex;

flag = true;

this.notify();

}

public synchronized void out() {

if (!flag) { this.wait(); } catch (Exception e) {}

syso(name + " " + sex);

flag = false;

this.notify();

}

}

r.out();

}

}

三、生产者消费者

一个线程生产一个 另一个线程消费一个

class Resource {

private String name;

private int count = 1;

private boolean flag = false;

public synchronized void pro(String name) {  //生产

if (flag) try { this.wait(); } catch (Exception e) {}  //同步函数的锁是this

this.name = name + count++;

syso(Thread.currentThread().getName() + "  生产  " + this.name);  //生产的商品带一个编号

flag = true;

this.notify();

}

public synchronized void cus() {  //消费

if (!flag) { this.wait(); } catch (Exception e) {}

syso(Thread.currentThread().getName() + "  消费  " + name);  //消费了哪个商品

flag = false;

this.notify();

}

}

class Cus implements Runnable {  //消费者

private Resource r;

Output(Resource r) {

this.r = r;

}

public void run(r) {

while (true) {

r.cus();

}

}

}

class Pro implements Runnable {  //生产者

private Resource r;

Input(Resource r) {

this.r = r;

}

public void run() {

while (true) {

r.pro("商品");

}

}

}

...

Resource r = new Resource();

new Thread(new Cus()).start();

new Thread(new Pro()).start();

但是在真实的开发中 都是很多个生产者 很多个消费者

目前我对多线程还没多少了解 也不知道什么实际开发中的栗子来说明这个

如果还用上面的那个程序 然后多开几个线程的话 就会出现 生产了多个 只消费了一个 生产了一个 被消费了多次的现象

这显然不是我们想要的

while循环

我们先来看看问题出在哪里

public synchronized void pro(String name) {  //①

if (flag) try { this.wait(); } catch (Exception e) {}  //②

this.name = name + count++;

当t1 t2两个生产者线程都wait以后 此时消费者唤醒了其中一个线程 这个被唤醒的线程 直接从它等待的地方②醒来 继续向下执行 尽管这时flag已经是true（已经有了一个生产了未被消费的商品）它没有经过①的判断就向下执行了

我们需要的是 每次唤醒之后 还要判断一下flag的状态

把if 改成while就可以解决这个问题 它醒来之后 还是去判断flag 如果已经有了未被消费的商品 就接着等待

notifyAll

但是把生产者消费者的if判断都改成while以后 会迎来新的问题 就是可能在某一刻 四个线程全部等待了 程序就死那了

因为唤醒的是本方的线程 本方的线程唤醒后还不满足条件 就继续等待 对方的线程也是如此 就都等待了

那就让唤醒的时候 全部唤醒 就不会出现这个问题了

this.notifyAll();

全醒了之后也没关系 因为都要进行判断的

整理一下 就是这样的

class Resource {

private String name;

private int count = 1;

private boolean flag = false;

public synchronized void pro(String name) {  //生产

while (flag) try { this.wait(); } catch (Exception e) {}  //同步函数的锁是this

this.name = name + count++;

syso(Thread.currentThread().getName() + "  生产  " + this.name);  //生产的商品带一个编号

flag = true;

this.notifyAll();

}

public synchronized void cus() {  //消费

while (!flag) { this.wait(); } catch (Exception e) {}

syso(Thread.currentThread().getName() + "  消费  " + name);  //消费了哪个商品

flag = false;

this.notifyAll();

}

}

 

concurrent.locks

虽然这样行了 但是还不够高效

既然本方的线程不满足条件 还唤醒他 那不是占资源了 其实只需要唤醒对方就可以

JDK1.5+里提供了更便捷的解决方案（在1.5之前 都在使用while和notifyAll）

java.util.concurrent.locks.Lock接口

Lock提供了比使用synchronized方法/语句可获得的更广泛的锁的操作 它替代了synchronized使用

lock()

unlock()

java.util.concurrent.locks.Condition接口

Condition替代了Object监视器方法的使用

await()  -  让当前线程处于等待状态

signal()  -  唤醒一个等待的线程

signal()  -  唤醒所有的等待线程

 

class Resource {

private String name;

private int count = 1;

private boolean flag = false;

private Lock lock = new ReentrantLock();  // 是Lock接口的实现类

public synchronized void pro(String name) {  //生产

while (flag) try { this.wait(); } catch (Exception e) {}  //同步函数的锁是this

this.name = name + count++;

syso(Thread.currentThread().getName() + "  生产  " + this.name);  //生产的商品带一个编号

flag = true;

this.notifyAll();

}

public synchronized void cus() {  //消费

while (!flag) { this.wait(); } catch (Exception e) {}

syso(Thread.currentThread().getName() + "  消费  " + name);  //消费了哪个商品

flag = false;

this.notifyAll();

}

}
