零、什么是hibernate

数据库操作框架

  • jdbc与hibernate的优缺点

1.jdbc

缺点

代码繁琐

要建立连接 建立Statement 要关闭Statement 要关闭连接...

面向数据库操作（而非面向对象）

做到了连接的缓存（连接池）但没有数据缓存（致命缺点）

当一次操作很多数据的时候 会很慢的

优点

灵活（因为比较底层）

比较适合优化（容易看清底层）

 

2.hibernate

优点（面试题）

代码简单

面向对象编程

有数据缓存（是世界上很牛的）

解耦（与数据库无关）

移植性比较强 比如要从MySQL移植到Oracle 分页不用改

缺点

没有办法干预SQL语句的生成

比如查cid从1到3的数据 可以有很多种方法 用hibernate就没得选了

SELECT * FROM classes WHERE cid in(1,2,3)

SELECT * FROM classes WHERE cid=1 OR cid=2 OR cid=3

SELECT * FROM classes WHERE cid BETWEEN 1 AND 3

SELECT * FROM classes WHERE 1<=cid<=3

SELECT * FROM classes WHERE cid=1 UNION SELECT * FROM classes WHERE cid=2 UNION SELECT * FROM classes WHERE cid=3

优化性能方面比较弱

如果有上千万级别的数据记录数 不适合用hibernate

小型系统/内部管理系统 适合使用hibernate

大项目不一定有大数据 比如如家的系统 全国连锁 项目很大 但一晚上就住几万几十万人 数据不大

 

  • hibernate是ormapping

ormapping（对象关系映射）

对象：java对象

关系：关系型数据库

映射：对象与数据库之间的关系（通过hibernate的映射文件）

表  -  类

字段  -  属性

字段类型  -  属性类型

表中的一行记录相当于一个java对象 表中的所有记录相当于对象的集合

我们称这个类为持久化类

 

  • hibernate的三要素

持久化类  -  产生对象存储数据

映射文件  -  建立表和类之间的关联

配置文件  -  数据库连接

 
一、第一个hibernate程序

1.导包

hibernate-3.5.6

2.配置文件

在src下 hibernate.cfg.xml

<hibernate-configuration>

<session-factory>  //一个 session-factory 代表一个数据库连接 但是只能连接一个

<property name="hibernate.connection.username">root</property>  //用户名

<property name="hibernate.connection.password">root</property>  //密码

<property name="hibernate.connection.driver_class">com.mysql.jdbc.Driver</property>  //连接数据库的驱动

<property name="hibernate.connection.url">jdbc:mysql://localhost:3306/database</property>  //url

<property name="hibernate.connection.dialect">org.hibernate.dialect.MySQLDialect</property>  //方言（告诉hibernate连接的数据库的类型）

<!--

当hibernate启动的时候对表的处理方式

validate  -  只检测表结构

update  -  发现没有表创建表 结构不符更新表

create  -  每次启动都创建新的

create-drop  -  每次启动都创建新的 每次关闭删除表

-->

<property name="hbm2ddl.auto">update</property>

<mapping resource="com/yutouxiuxiu/.../Person.hbm.xml" />  //引入映射文件

<property name="show-sql">true</property>  //在控制台打印正执行的sql语句

<property name="format-sql">true</property>  //在控制台打印正执行的sql语句 （有好看格式的）

</session-factory>

</hibernate-configuration>

3.持久化类

public class Person implements Serializable {  //必须实现Serializable接口 jvm可以将其变成二进制码 该类创建出来的对象 就可以在网络中传输

//必须提供无参的构造函数

private Long pid;

private String name;

getter/setter...

}

4.映射文件

放在Person的同包中 Person.hbm.xml

用hibernate mapping editor打开

<hibernate-mapping>

<!--

class 描述持久化类

name  -  类的全名

table  -  表名（不写默认是类名）

catelog  -  数据库名（其实在配置文件中已经配置过了 不用写）

-->

<class name="....Person" table="person">

<!--

id 标示符属性（告诉它哪个是主键）

name  -  类中的“主键”（来自持久化类的属性名 而不是set和get方法名）

length  -  规定该持久化类对应的表中的字段的长度

column  -  在表中对应的字段（不写默认和name一致）

type  -  属性的数据类型

-->

<id name="pid" length="8" column="pid" type="java.lang.Long">

<!--  generator主键生成策略  -->

<generator class="increment"></generator>

</id>

<!--  property 一般属性  -->

<property name="name" length="" column="name" type="java.lang.String"></property>

</class>

<hibernate-mapping>

5.生成表

hibernate可以根据持久化类和映射文件生成表

Configuration conf = new Configuration();  //创建对象用于解析配置文件

conf.configure();  //加载配置文件

//conf.configure("");  //从指定路径加载配置文件 任意路径 任意名称

conf.buildSessionFactory();

6.save操作 插入记录

Configuration conf = new Configuration();  //创建对象用于解析配置文件

conf.configure();  //加载配置文件

SessionFactory sessionFactory = conf.buildSessionFactory();  //见下面

Session session = sessionFactory.openSession();

Transaction tran = session.beginTransaction();  //开启事务 如果不是查询必须开启事务才行

Person p = new Person();

//p.setPid();  //不用写 它自动生成

p.setName("芋头");

session.save(p);

tran.commit();  //提交事务

session.close();  //关闭session

 

二、SessionFactory

（面试题）

1.存放配置

hibernate配置文件描述的信息全部在SessionFactory中存放

数据库连接信息

映射文件信息

其他的信息

2.存放映射

所有的映射文件信息、持久化类信息都在SessionFactory中

3.是重量级的类 共享数据

存放了很多信息 而且信息都是共享的

4.本身是数据安全的

线程安全的

 

三、Session

1.Session是由SessionFactory产生的

2.hibernate中增删改查是由Session完成的

3.打开连接 完成对数据库的操作

 

四、hibernate的执行流程

1.创建Configuration对象

Configuration conf = new Configuration();

2.利用Configuration加载配置文件

两种方式

conf.configure();  //加载src下固定名称的配置文件

conf.configure("");  //从指定路径加载配置文件 任意路径 任意名称

3.加载配置文件中配置的映射文件

4.产生SessionFactory

SessionFactory sessionFactory = conf.buildSessionFactory();

5.产生Session

Session session = sessionFactory.openSession();

6.开启事务（如果要查询 就不需要开启事务）

Transaction tran = session.beginTransaction();  //开启事务

7.进行增删改查

8.提交事务（如果是查询 没有事务 就不用提交）

tran.commit(); 

9.关闭Session

session.close();

 

五、用hibernate完成增删改查

HibernateUtils

public static SessionFactory sessionFactory;

static {

Configuration conf = new Configuration();

conf.configure();

SessionFactory sessionFactory = conf.buildSessionFactory();

}

 

Dao extends HibernateUtils

 

  • 改

Session session = sessionFactory.openSession();

Transaction tran = session.beginTransaction();

//get(持久化类类型, Serializable 主键值)

//Serializable：因为所有基本类型的包装类都实现了这个接口 所以它是这个类型的

 

//方式1

//必须把对象拿出来 然后才能改 必须把它先变成对象 才能面向对象操作 所以必须要把它先弄出来 这种方式只能按照主键取对象

//hibernate的思想是 把数据库中的东西变成对象 然后操作对象 然后再弄回去

Person person = (Person)session.get(Pserson.class, 1L);  //取cid为1的数据从数据空中提取出来变成对象

person.setName("修修");  //set了什么 就改什么 其余的不变

session.update(person);

 

//方式2 不推荐

//但是如果不set别的 别的就被改成null了

Person person = new Person();

person.setPid(1L);  //虽然Person对象是新创建的 但是只要设置了主键 hibernate就执行update操作 而不是insert

session.update(person);

 

tran.commit();

session.close();

 

  • 删

Session session = sessionFactory.openSession();

Transaction tran = session.beginTransaction();

Pserson person = session.get(Person.class, 1L);

session.delete(person);

tran.commit();

session.close();

session.close();

 

六、hibernate的内部运行原理

session的save方法中传入的是持久化对象 那么就能找到对应的持久化类

根据持久化类找到映射文件

找到表名

找到字段名和属性名的对应关系

通过get方法得到里面的内容

 

七、主键生成器

increment  -  在最大值上+1

主键必须是数字类型 它会先查出表中主键的最大值 然后+1

即使自己手动setId 仍然按照你设置的主键生成策略 hibernate根本就不看

identity  -  让它依靠数据库完成自增长

需要在数据库表中设置成自增长 auto_increment

assigned  -  手动生成

需要自己手动设置 setId

native  -  hibernate自己选

sequence  -  序列

仅支持Oracle

uuid  -  UUID

企业里一般都用这个

 

八、类型

java类型（比hibernate类型效率高 一般使用java类型）

java.langString  -  varchar/varchar2

java.lang.Lang  -  long

......

hibernate类型（它得先找java类型）

string  -  varchar/varchar2

integer,long,float,double,string,true_false,yes_no

......
