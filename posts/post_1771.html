jBPM - 流程实例管理

{ post:time }

流程实例管理 使用 ExecutionService

一、启动流程实例

对于采购流程

1.启动实例

@Test

public void qidong() {

ProcessEngine processEngine = new Configuration().buildProcessEngine();

//流程实例管理 使用 ExecutionService

ExecutionService executionService = processEngine.getExecutionService();

 

//根据pdid启动

executionService.startProcessInstanceById("caigou-1");

 

//根据pdkey启动 如果有相同key 启动最新版本流程的实例

executionService.startProcessInstanceByKey("caigou");

}

2.启动实例之后数据表的变化

jbpm4_execution 正在运行的流程实例表

每启动一个实例 当这个实例正在运行中 就对应execution表的一行数据

pid 流程实例id 对应表中的ID属性

如果实例结束了 就被从这个表中移除了

jbpm4_hist_procinst 流程实例历史记录表

启动实例的同时 会在这里也插入一条记录

jbpm4_task 正在运行的任务表

流程已启动 就马上进入了第一个任务节点 流程实例启动 就会把第一个任务创建在这里

如果第一个活动节点 不是任务 就只会在活动节点表中创建记录 不会在任务表中创建记录

任务执行完毕就会被删除

jbpm4_hist_task 任务历史记录表

和流程实例历史表类似 这个是存历史任务的

jbpm4_hist_actinst 活动历史记录表

jBPM流程中有多重活动节点 任务是活动的一种 

 

3.控制流程实例向后流转

@Test

public void liuzhuan() {

ProcessEngine processEngine = new Configuration().buildProcessEngine();

//流程实例管理 使用 ExecutionService

ExecutionService executionService = processEngine.getExecutionService();

 

//流程实例的ID

//流转的name属性（xml文件的transition标签的name属性 如果没有name属性 才不用指定）

signalExecutionById("caigou.3001", "to 总经理审批");

//signalExecutionById("caigou.3001", "to end1");

}

 

4.中止流程

@Test

public void liuzhuan() {

ProcessEngine processEngine = new Configuration().buildProcessEngine();

processEngine.getExecutionService().endProcessInstance("caigou.40001", ProcessInstance.STATE_ENDED);

}

可以从该属性看出 这个流程实例是在半路上结束的

5.查询所有正在运行的流程实例

createProcessInstanceQuery().list();
