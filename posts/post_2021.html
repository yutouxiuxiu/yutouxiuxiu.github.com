一、多态

1.有个小栗子

在java中

有一个手电筒类 里面有点亮方法 接收红色玻璃类型的参数

有一个蓝色玻璃对象 有一个红色玻璃对象

只能给手电筒安红色玻璃 因为点亮方法不能接收其他颜色玻璃的参数

java用多态解决了这个问题

定义一个玻璃类 各种颜色的玻璃继承这个类

手电筒的点亮方法接收玻璃类型的参数

这样可以在点亮的时候安装各种颜色的玻璃

 

2.java中为什么需要多态？

因为java是强类型语言

参数声明的类型 传参就必须是该类型

所以 在声明参数类型的时候 声明他们的父类 传的时候 传子类 就很灵活

 

3.php中为什么需要多态？

但是php是弱类型语言 太灵活 太变态了

参数不用声明类型 传什么都行

比如说 有一个猪的对象 因为php是弱类型的 可以将猪这个对象作为参数传递到手电筒的点亮方法中

然后一打开手电筒 一头猪就掉地上了 很不合理

我们不喜欢让它这样灵活

 

class Glass {
}

class RedGlass extends Glass {
    public function shine() {
        echo '红光';
    }
}

class Torch {
    public function on(Glass $Glass) {
        $Glass->shine();
    }
}

class Pig {
    public function shine() {
        echo '傻气闪耀';
    }
}

$red = new RedGlass();
$pig = new Pig();
$torch = new Torch();

$torch->on($red);
$torch->on($pig);

/* 结果
红光

*/
