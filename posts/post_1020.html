Action与ServletAPI

{ post:time }

如何在Action的execute方法中获取使用request，response，session，application对象

 

一、解耦方式

它不是用来获取request，response，session，application对象的

是获取这些域对象的解耦对象（Map）的

对域对象的Map进行操作 就等同于操作域对象一样

优点：与ServletAPI没有耦合

缺点：没有办法完全操作ServletAPI

    不能调用方法 只能操作域对象中的域功能

    例如不能request.getContextPath(); 只能添加删除获取Attribute

1.request域

//1.获取ActionContext

ActionContext ac = ActionContext.getContext();

//向request域中添加内容

ac.put(“requestKey”, “requestValue”);

//获取request域中的内容

ac.get(“requestKey”);

//获取request的Map 对这个Map操作等同于操作request域

Map<String, Object> request = ac.getContextMap();

…

2.session域

//获取session的解耦Map

Map<String,Object> sessionMap = ac.getSession();

sessionMap.put(“sessionKey”,”sessionValue”);

…

3.application域

Map<String,Object> applicationMap = ac.getApplication();

…

4.获取请求参数

Map<String,Object> params = ac.getParameters();

//注意 返回的类型是String类型的数组

String username = ((String[]) params.get(“username”))[0];

5.获取值栈

ac.getValueStack();

 

二、感知接口方式

基本不用

需要implements ServletRequestAware, ServletResponseAware, ServletContextAware接口

里面有一个未实现的方法

private HttpServletRequest request;

private HttpServletResponse response;

private HttpServletContext context;

@Override

public void setServletRequest(HttpServletRequest request) {

    this.request = request;

}

@Override

setServletResponse(HttpServletResponse response) {

    this.response = response;

}

@Override

setServletContext(HttpServletContext context) {

    this.context = context;

}

Struts会调用这些方法把ServletAPI传递给我们

会在execute方法执行之前执行这些方法

这样在execute方法中使用 就没问题了

 

不过这里Struts把这些域对象进行了增强 和原来的不一样了

 

三、ServletActionContext方式

经常使用

HttpServletRequest request = ServletActionContext.getRequest();

HttpServletResponse response = ServletActionContext.getResponse();

HttpServletContext context = ServletActionContext.getServletContext();

就这么用就行了
