3.让一个Servlet有多个处理方法

以前一个Servlet只能处理一个请求 当网站功能很多的时候 Servlet会暴增

编写方式

public class UserServlet extends BaseServlet {    //1.继承BaseServlet

    //不再需要doPost，doGet

    //2.返回值用String

    //3.方法名写成具体功能的名称

    public String add(HttpServletRequest request, HttpServletResponse) throws ServletException, IOException {

        //4.返回值为转发到的位置

        return “/index.jsp”;

    }

    public String edit(HttpServletRequest request, HttpServletResponse) throws ServletException, IOException {

        //这里可以手动重定向

        return null;    //4.不想转发请返回null

    }

}

访问方式

访问的时候需要传递参数

UserServlet ?method=add

原理

BaseServlet

先获取参数method的值

按照method查找本类中对应的方法

不存在则抛异常

然后用反射invoke调用该方法 得到返回值

返回值不null不空 就转发
