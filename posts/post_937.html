  • 继承

   is a 谁是谁的一种
  好处
    1.提高了代码的复用性
    2.让类与类产生了关系 有了这个关系才有了多态的特性

（注意！！！）只支持单继承 不支持多继承（c++支持多继承）
    多继承容易带来安全隐患 多个父类中有同样的方法 它就不知道运行哪个了
    java用多实现形式来表现多继承
   
    想要用一个体系 先看父类 再看子类
    查阅父类功能 创建子类对象 （父类也许不让创建对象 比如 抽象类 接口）

  • 继承出现后 类成员的特点


    1.变量  构造函数
        */

        class Fu {
            int num = 4;
        }

        class Zi extends Fu {
            int num = 5;
        }

        /*
        打印num为子类的5
            可以打印 this.num 子类的num
                this 本类对象的引用
                如果子类李没有 只有父类里有 用this也能访问
            可以打印 super.num 父类的num
                super 父类对象的引用
     */    
    /*
    2.函数
        子类出现和父类一样的函数时 会对父类方法进行重写覆盖
        当子类继承了父类 沿袭了父类的功能到子类中 但是子类虽具备该功能 但功能的内容 却和父类不一致
        这时 没有必要定义新功能 而是使用覆盖特性 保留父类的功能定义 并重写功能内容
        如果有一部分用过原来的函数 就用 super.函数名();
       
        （注意！！！）  子类权限必须大于等于父类权限才能覆盖（父类的是private 不叫覆盖 因为子类都不知道父类有 就是一个函数 不是覆盖）
                        静态只能覆盖静态
                       
                        重载：只看同名函数的参数列表
                        重写：子父类方法一模一样
                                如果父类中有一个 int method() 子类中有一个 void method() 这样不行 无法覆盖 编译不通过
   
    3.构造函数
        构造函数不能覆盖 因为子父类的构造函数名都不一样
       
        会先执行父类的构造函数 再执行子类的构造函数
        因为子类所有的（注意！！！是所有的）构造函数中的第一行 有一条隐式的 super(); -> 会访问父类中空参数的构造函数
        （注意！！！）如果显示指定 就执行显式指定的 如果父类中没有空参数的构造函数 就必须显式指定
        （显式指定必须在第一行 显式指定的 有可能是this()也有可能是super() 子类中至少有一个构造函数会访问父类中的构造函数）
        子类必须要先访问父类的构造函数 因为子类使用之前 要看看父类是怎么对一些东西初始化的
       
        父类的构造函数也有super(); 它访问了Object的构造函数
    */

class Fu {
public Fu() {
System.out.println(this);
}
}

class Zi extends Fu {

}

new Zi();

/*
打印的是Zi...
因为类的加载过程
1.加载Zi到内存
2.加载Fu到内存
3.执行Zi构造方法
4.执行Fu构造方法
*/

  • final 关键字


    1.可以修饰类 变量 函数
    2.被final修饰的类 不可以被继承（继承打破了封装性 要是不想被别人修改的 就定义final）
    3.被final修饰的方法 不能被覆写（有些访问底层 内部实现之类的不想被覆写的 可以定义final）
    4.被final修饰的变量 是常量 只能赋值一次 既能修饰成员变量 又能修饰局部变量
        常量就得所有字母大写 单词间用下划线
        如果在成员位置上的时候 还加一个static
            public static final ... 全局常量
