防止SQL注入

{ post:time }
零、引例

在用户登录的时候 会执行这样的语句 如果查找到了记录 就说明用户名密码正确

select * from user where username=’张三丰’ and password=’123’

注入手段

1.传入 张三丰’#

select * from user where username=’张三丰’#’ and password=’’

由于#是sql的注释 后面会被当做没用的内容

select * from user where username=’张三丰’

3.传入 张三丰’ or ‘1=1

select * from user where username=’张三丰’ or ‘1=1’ and password=’’

1=1永远成立 所以可以查找到记录

 

一、用PreparedStatement 防止注入漏洞

用PreparedStatement 预编译传输对象

是Statement的子类

它会先把sql语句编译成机器码 而不是直接把sql语句传过去

//…

//不用之前的Statement了

PreparedStatement ps = conn.preparedStatement(“select * from user where username=? and password=?”);    //这个是sql语句的骨干 传进来的参数用?替代

//传入参数

ps.setString(1, user.getUsername());

ps.setString(2, user.getPassword());

//执行

ResultSet rs = ps.executeQuery();

//…

好处1：

这样 它就先把sql语句的主干编译好了 后面传入的参数只按照参数处理

而不能被当做sql语句的主干处理 就不会发生注入的问题了

好处2：

预编译过的 效率高

好处3：

写sql语句的时候不用拼凑字符串
