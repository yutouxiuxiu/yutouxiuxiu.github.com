jBPM - 流程变量管理

{ post:time }

流程在流转的过程中 办理任务的时候 会产生一些业务数据 这些业务数据需要和jBPM流程实例相关联

关联到流程实例的变量 就称为流程变量

支持的类型

后三种是复杂的自定义对象 long类型/string类型的主键对应的PO对象（hibernate实体对象）

实现serializable接口的序列化对象

 

一、流程变量的读写

1.在流程实例启动时 保存流程变量

ProcessEngine processEngine = new Configuration().buildProcessEngine();

ExecutionService executionService = processEngine.getExecutionService();

//变量的Map集合

Map<String, Object> vars = new HashMap<String, Object>();

vars.put("company", "Yuxiu Home");

//启动实例 并保存流程变量

executionService.startProcessInstanceByKey("caigou", vars);

就会在 jbpm4_variable 表创建一条记录

 

2.使用ExecutionService 结合 pid 读写变量

ProcessEngine processEngine = new Configuration().buildProcessEngine();

ExecutionService executionService = processEngine.getExecutionService();

 

//读

executionService.getVariable("caigou.80001", "company");

//写

executionService.setVariable("caigou.80001", "weather", "晴");

 

3.通过 TaskService 结合 任务id 进行变量的读写

ProcessEngine processEngine = new Configuration().buildProcessEngine();

TaskService taskService= processEngine.getTaskService();

 

//读

taskService.getVariable("80003", "weather");

//写

Map<String, Object> vars = new HashMap<String, Object>();

vars.put("time", "今天");

taskService.setVariables("80003", vars);
