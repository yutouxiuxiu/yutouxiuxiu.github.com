五、指令元素

有三个指令 指示JSP的特定行为 使其执行某些步骤

<%@ 指令名 属性名=”属性值”[, 属性名称=”属性值”…] %>

1.page指令

用来指定JSP文件的全局属性

<%@ page … %>

属性：

language=”java”

  -  SUN公司推出JSP时 还想以后JSP里不仅仅可以写JAVA代码 可以配置里面写什么 但是现在还没有其他的

extends=”org.apache.jasper.runtime.HttpJspBase”

  -  JSP被翻译成Servlet的时候要继承哪个类（HttpJspBase 它就继承了 Servlet） 这里的值必须是HttpServlet 或其子类

session=”true/false”

  -  默认是true 这时可直接使用session 上行的代码 可以替代下行代码 否则获取session只能写下行

session.getAttribute(“”);

request.getSession().getAttribute(“”);

import

  -  导包 导入java类的

      每个属性都可以单独写成一个 也可以直接放在其他的page里 这个属性可以出现多次 其他的一般只能出现一次

<@page import=”java.util.List”%>

      java.lang.*，javax.servlet.*，javax.servlet.http.*，javax.servlet.jsp.* 是默认被导入的 不用自己导入

buffer=”8kb” 和 autoFlush=”true”

  -  向页面输出的缓冲区 默认是8KB（8192）

      生成的java源代码如下

pageContext = _jspxFactory.getPageContext(this, request, response, null, true, 8192, true);

errorPage

  -  处理错误的友好页面 让它出错误的时候不显示那些错误 让它友好提示

errorPage=”错误处理页面.jsp”

     在企业开发里 一般出错的时候不这样做 在每个页面都得配置 麻烦 在项目的web.xml里配置通用的错误页面

     errorPage 属性 优先于web.xml中的配置

<error-page>

    <error-code>404</error-code>

    <location>/404.jsp</location>

</error-page>

<error-page>

    <error-code>500</error-code>

    <location>/500.jsp</location>

</error-page>

isErrorPage=”false”

  -  如果是false 翻译成的java代码里面会有8个对象 为true时 多了个 excrption 可以获取错误信息等相关内容

     这9个称为内置对象 在下面有介绍

exception.getCause();

contentType

  -  浏览器的默认打开编码方式

pageEncoding

  -  JSP生成Servlet 以及保存的时候采用的编码

isELIgnored=”false”

  -  页面是否忽略EL表达式 默认支持EL表达式 被忽略时EL表达式就直接当成文字被输出了

8.out（JspWriter）

可以将页面上的内容输出到浏览器

方法 返回值类型 方法说明
clear() void 清除网页上输出内容
clearBuffer() void 清除缓冲区内容

println(String str) void 进行页面输出并换行

和response.getWriter()的区别

response.getWriter() 返回 PrintWriter

out 的对象时 JspWriter

举个栗子：

JSP代码

<%=”a<br/>”%>    会被翻译成out（JspWriter）

<% response.getWriter().println(“b<br/>”); %>    是PrintWriter

<% out.println(“c<br/>”)%>    是JspWriter

输出结果

b

a

c

因为response和out都有自己的缓冲区 out缓冲区的内容会先被写道response缓冲区的最后面 然后再由response输出到页面

七、标签（动作）

<jsp:…/>

注意标签要有结束的斜杠 或者</jsp:…>

也叫做JSP的动作

可以使用一个标签来代表一段JAVA代码 可以简化代码

1.动态包含

<jsp:include file=”.jsp”/>

实现过程：

主jsp被翻译成java代码时 包含的文件也会生成一个Servlet 解析到这个动态包含标签的时候 会把包含的jsp对应的Servlet运行的结果包含过来

（注意！！！）这里包含的文件可以定义重复的变量 因为每个文件都会生成各自的Servlet 就不能简单的跨越文件使用变量了

动态包含用param标签传递参数 详见下面

问题：

Q：可以直接包含Servlet么？

A：不可以 都是包含页面

      静态包含还可以包含其他文件 比如html，txt

      动态包含只能是jsp 因为要生成Servlet

2.转发

<jsp:forward page=”跳转去的路径”/>

相当于 request.getRequestDispatcher(“”).forward(request, response);

3.参数

（1）在forward标签内 向跳转去的文件传参数

<jsp:forward page=”跳转去的路径”>

    <jsp: param name=”” value=””/>（由于表情识别所以在冒号后加了空格 实际没有）

</jsp: forward>

        在跳转去的文件里用request.getParameter(“”)接收

        一般不这么用 一般用request.setAttribute传递

（2）在include标签内 向动态包含的文件传参数

<jsp:include file=”动态包含的文件”>

    <jsp: param name=”” value=””/>（由于表情识别所以在冒号后加了空格 实际没有）

</jsp:file>

        在动态包含的文件里用request.getParameter(“”)接收

4.和JavaBean相关的标签

<jsp:useBean/>

<jsp: setProperty/>

<jsp:getPeoperty/>

JavaBean和内省的相关内容 点击这里查看 ?p=427

作用：

可以把从表单提交到这个jsp的数据 轻松简单的封装到描述这些数据的对象中 而不用get很多的parameter然后再调用数据封装类（JavaBean）的很多set方法

这些标签可以把和数据封装类属性名相对应的表单元素的name值提交过来的值 封装到这个数据封装类中

如果不是提交到jsp 是提交到一个servlet 就要用内省技术来方便地把这些数据存到数据封装类的对象里

举个栗子：

要把POST提交到jsp的数据封装到JavaBean里（详见JavaBean和内省 上面有链接）

使用传统的方式：

Person p = new Person();

String name = request.getParameter(“name”);

String age = request.getParameter(“age”);

p.setName(name);

p.setAge(age);

使用标签：

<jsp:useBean id=”p” class=”cc.vicp.yutouxiuxiu.family.Person” scope=”page”/>

<jsp: setProperty name=”p” property=”name” />

<jsp: setProperty name=”p” property=”age” />    （注意！！！） 这里的peoperty 一定是提交表单数据的name和JavaBean里的属性名一致

        后两句也可以用简写

<jsp: setProperty name=”p” property=”*” />   只要对应的能找到的 都自动装进去

 

<jsp:useBean/>

属性：

id  -  对象的唯一标识

class  -  JavaBean的类对象

scope  -  对象所存的范围（什么时候没有）

               page/request/session/application

 

<jsp: setProperty/>（由于表情识别所以在冒号后加了空格 实际没有）

属性：

name - useBean里设置的id

property - 属性名（提交表单数据的name和JavaBean类里的属性名 和这个 都要一致）

value
