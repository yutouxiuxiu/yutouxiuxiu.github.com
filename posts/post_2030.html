php面向对象 - self/parent/final

{ post:time }
一、self/parent

1.self

引用本类的静态属性/静态方法

self::静态属性  -  在类内调用本类的静态属性

self::静态方法()  -  在类内调用本类的静态方法

new self()  -  在类内产生本类对象

2.parent

引用父类的属性/方法

parent::静态属性  -  在子类中调用父类的静态属性

parent::方法()  -  在子类中调用父类的方法

 

在继承了父类的方法时 使用this和parent的区别

class Fu {

public function a() {

echo 'a()';

}

}

class Zi extends Fu {

public function b1 {

parent::a();

}

public function b2 {

$this->a();

}

}

在上面的栗子中 效果是一样的 从效率角度来讲 "parent::"稍微快一点点 因为用this 是先从本类找 找不到再去父类找

但是从面向对象的角度来说 继承过来就是自己的 this更符合面向对象的思想

并且在继承结构很深的时候 如果用parent 就很不方便 需要写很多层 this一下就搞定了

class A {}

class B entends A {}

...

class F extends E {

... parent::parent::parent...

}

如果是私有的属性和方法 用parent 就可以访问

 

二、final

可以修饰类名、方法名、不能修饰属性  -  和java不同 java中可以修饰属性 为常量

final的类 不能继承

final的方法 不能覆盖
