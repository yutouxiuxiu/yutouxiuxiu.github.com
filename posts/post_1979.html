一、环境搭建

1.Maven项目环境搭建

（1）建立Maven工程

elipse的m2eclipse插件，myeclipse自带  -  新建Maven项目 -> Create a Simple Project

项目坐标  -  groupId，artifactId，version

项目打包方式  -  jar（被别的项目引用的）, pom（被别的项目依赖的）, war（可被部署的ee项目）

编译版本  -  1.6

目录结构  -  src/main/java，src/main/resources，src/test/java，src/test/resources，src/main/webapp

（2）使用坐标导入依赖

http://search.maven.org

http://mvnrepository.com

struts2 15.2, hibernate 3.6.10, spring 3.2, jpa（java持久化接口 hibernate是jpa的实现）, log4j, c3p0, mysql, junit, poi, pinyin4j, hessian, hibernate search(lucene), ik分词器, jbpm, javamail, ehcache

（3）常用命令

mvn clean

mvn package

mvn test

mvn build

mvn tomcat:run  内嵌

mvn tomcat:deploy  部署到外部

tomcat-maven-plugin 插件

（4）项目分解

很多企业中按照表现层、业务层、持久层 进行项目分解

通过依赖的方式来引用和运行

二、完整流程

1.搭建开发流程规范

（1）使用经典三层结构

每层之间面向接口整合 层内部公用重复代码 使用抽象类进行抽取和复用

Action extend BaseAction -> Service接口 -> ServiceImpl extends BaseService implements 业务接口 -> GenericDao接口

使用xml配置 使用注解注入

现在企业中使用注解进行配置的越来越多 提高效率

applicationContext按照三层分解

（2）通用Dao组件 面试说

基于反射和泛型 通用增删改查接口 分页查询接口 索引库查询接口

QBC 命名查询（HQL，SQL）

2.登录

验证码

struts2 表单校验

国际化信息显示

登录拦截器

3.将页面放在WEB-INF下

通过Action来访问 以便于拦截器进行拦截
