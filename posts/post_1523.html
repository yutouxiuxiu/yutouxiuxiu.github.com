1.创建数据库

（1）MySQL有数据库的概念 每个项目建一个database 建一个独立用户 只能操作当前数据库

create database bos charset utf8;

如果不指定字符集 会使用character_set_database的字符集

character_set_server 如果它也设置成utf8 向数据库中插入就不会有乱码 如果是拉丁1 就完了

my.ini文件中 [mysqld]中有character_set_server 把它=utf8就行了

 

新建用户

create user username[@localhost | @'%'] identified by 'password';

分配权限

grant all on bos.* to 用户名

 

（2）Oracle一个库 就是一个服务 硬盘分配1G空间 存在table space（表空间）的概念 用户的权限是对应表空间的 每个项目对应一个单独的表空间 创建一个用户 具有表空间的使用权限

三、设计通用DAO组件

设计dao层 传统方式 是设计一个BaseDao 对每一个表创建一个Dao继承BaseDao

在《Spring3.x企业应用开发实战》的 17.2.6持久层设计 里有详细的介绍

 

其实完全可以用一个类就可以完成所有的增删改查

com.yutouxiuxiu.bos.dao.GenericDao.class

public interface GenericDao<T> {

public void save(T obj);

public void update(T obj);

public void delete(T obj);

public List<T> findAll();

public T findById(Serializable id);

// 条件查询

public List<T> findByNamedQuery(String queryName, Object... args);  //命名查询 把HQL语句放到配置文件中 然后service通过配置文件中HQL语句的命名进行查询

public List<T> dindByCriterial(DetachedCriterial detachedCriterial);  //离线对象查询 通过面向对象的方式写条件 然后通过对象产生SQL语句

}

 

com.yutouxiuxiu.bos.dao.impl.GenericDaoImpl.class

public class GenericDaoImpl<T> extends HibernateDaoSupport impelements GenericDao<T> {

private Class clazz;

public GenericDaoImpl(Class<T> clazz) {  //为了查询的时候 能按照对应的类进行查询 因为泛型只在编译时期 泛型之后就擦除了 所以要传进class

this.clazz = clazz;

}

 

@Override

 

public void save(T obj) {

this.getHibernateTemplate().save(obj);  //spring提供的模板工具类简化写法

//this.getSession().save(obj);  //hibernate原始写法

}

​@Override

public void update(T obj) { ... }

​@Override

public void delete(T obj) { ... }

 

​@Override

public List<T> findAll() {

return this.getHibernateTemplate().find("FROM " + clazz.getName());

}

 

​@Override

public T findById(Serializable id) {

this.getHibernateTemplate().get(clazz, id);

}

 

​@Override

public List<T> findByNamedQuery(String queryName, Object... args) {

this.getHibernateTemplate().findByNamedQuery(queryName, args);

}

 

​@Override

public List<T> dindByCriterial(DetachedCriterial detachedCriterial) {

this.getHibernateTemplate().dindByCriterial(detachedCriterial);

}

}

public String editpassword() {

User loginUser = ServletActionContext.getResult().getSession().getAttribute("user");

try {

userService.editPassword(user);

public void editPassword(User user) {

//不能用update！直接用 就把所有的都改了 先把它查出来 再设置密码

persistUser.setPassword(user.getPassword());

//这个方法被事务管理了 事务提交 就改成功了

}
