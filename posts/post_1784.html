jBPM - 流程任务管理

{ post:time }

在实际的开发中 对于流程中的task节点 通常不会使用singalExecutionById

任务节点 实际上是需要人工参与来办理完成的 通常需要在页面上提交表单

可以使用 TaskService提供的API

 

在设计业务流程时 为流程中的任务节点指定办理人

先采用assignee 单一负责人

 

一、查询任务

@Test

public void chaxun(){

ProcessEngine processEngine = new Configuration().buildProcessEngine();

//使用 TaskService

TaskService taskService= processEngine.getTaskService();

 

//使用TaskQuery 查询所有任务

TaskQuery taskQuery = taskService.createTaskQuery();

List<Task> list = ​taskQuery.list();

for (Task task : list) {

task.getId();

task.getName();

task.getAssignee();

}

 

//查询个人任务

List<Task> list = ​taskService.findPersonalTask("销售人员");  //传入的是负责人名 和xml中assignee对应

}

 

二、办理任务

其实和流程向后流转效果一样 但是任务处理只能让任务向后流转 任务办理更强调是人来参与 向后流转是通用的一种东西 比如state节点 就不需要人来参与 就用signalExectionById来流转

 

四种ID

发布编号 jbpm4_deployment 的 dbid

流程定义编号 jbpm4_

流程实例编号 jbpm4_

任务编号 jbpm4_task 的 dbid

 

1.办理任务

@Test

public void (){

ProcessEngine processEngine = new Configuration().buildProcessEngine();

//使用 TaskService

TaskService taskService= processEngine.getTaskService();

 

//办理任务

taskService.completeTask("60009");    //jbpm4_task 表中的DBID

}
