/* 修改 提供的服务名称 命名空间（暴露了服务端的包结构 很有必要改） */
@WebService(serviceName="GreetingService", targetNamespace="http://www.yutouxiuxiu.com")
public class Hello {
    /* 修改 提供的方法名 */
    @WebMethod(operationName="sayHi")
    /* 修改 返回值名称 */
    @WebResult(name="greetingSentence")
    public String sayHello(
            /* 修改 参数名称 */
            @WebParam(name="name")
            String name,
            @WebParam(name="gender")
            int gender) {

    }
    
    public static void main(String[] args) {
        Endpoint.publish("http://192.168.1.194:6789/hello", new Hello());

MyBatis 使用注解开发

2014-02-11 14:31
一、基本

1.查询

创建一个Dao接口 在接口中写查询的方法体 然后在方法上加上注解 这样 调用这些方法的时候 就会被MyBatis接管 进行数据库的操作

public interface PersonDao {
    
    // 查询单个
    @Select("select * from person where person_id = #{personId}")
    @Results({    // Results注解只能做基本的字段映射
        @Result(property="id", column="person_id", id=true),
        @Result(property="name", column="name"),
        @Result(property="birthday", column="birthday"),
        @Result(property="address", column="address")
    })
    public Person selectPersonById(int personId);
    
    // 查询集合
    @Select("select * from person where name = #{name} and salary = #{salary}")
    @Results({
        @Result(property="id", column="person_id", id=true),
        @Result(property="name", column="name"),
        @Result(property="birthday", column="birthday"),
        @Result(property="address", column="address")
    })
    public List<Person> selectPersonByParams(Map<String, Object> map);
}

在调用的时候有两个关键的步骤

public class DaoTest {

    }
}

三、动态SQL

    // 使用一个java类来提供动态的SQL type - 类, method - 方法
    @SelectProvider(type=SQLHelper.class,method="getSQL")
    @Results({
        @Result(property="id", column="person_id", id=true),
        @Result(property="name", column="name"),
        @Result(property="birthday", column="birthday"),
        @Result(property="address", column="address")
    })
    public List<Person> selectPersonByCondition(Map<String, Object> map);

 

public class SQLHelper {
    public String getSQL(Map<String, Object> map) {
        String name = (String) map.get("name");
        Date birthday = (Date) map.get("birthday");
        Integer salary = (Integer) map.get("salary");
        
        SqlBuilder.BEGIN();    // 开始拼接
        SqlBuilder.SELECT("*");    // 拼接查询的字段
        SqlBuilder.FROM("person");    // 拼接表名
        if (name != null)
            SqlBuilder.WHERE("name like '%" + name + "%'");    // 在SqlBuilder中 不能使用${}
        if (birthday != null)
            SqlBuilder.WHERE("birthday < #{birthday}");    // 在SqlBuilder中 可以使用#{}
        if (salary != null)
            SqlBuilder.WHERE("birthday < #{salary}");
        
        return SqlBuilder.SQL();
    }
}

 

MyBatis 缓存

2014-02-11 11:48
一、一级缓存

不需要做任何的配置 是session级别的缓存 在同一个session范围之内可以使用缓存

1.一级缓存的原理

在一个session中 将查询的结果集缓存起来 以select的id为标识

再次查询的时候 会到缓存中查询结果是否存在 如果存在就不再发出SQL查询了 而是直接利用缓存中的数据

2.栗子

（1）同一session中

    @Test
    public void cache1() {
        // 获得session
        SqlSession session = sqlSessionFactory.openSession();
        try {
            // 发送SQL语句
            Person person = session.selectOne("com.yutouxiuxiu.Person.selectPersonById", 1);
            // 不发出SQL语句
            person = session.selectOne("com.yutouxiuxiu.Person.selectPersonById", 1);
        } finally {
            session.close();
        }
    }

（3）刷新缓存

在一个session中 如果第一次查询之后 又对数据进行了更改 就会将缓存做过期处理 第二次查询的时候 发现缓存是过期的 所以还要重新查询

以上代码会发出一个查询 一个更新 一个查询语句

 

二、二级缓存

二级缓存是sessionFactory级别的 在同一个sqlSessionFactory可以跨session使用缓存

1.开启二级缓存总开关

在核心配置文件中

<configuration>
    
    <settings>
        
        <!-- 开启二级缓存 -->
        <setting name="cacheEnabled" value="true" />

2.开启单个映射文件的二级缓存

<mapper namespace="com.yutouxiuxiu.Person">
    
    <!-- 开启当前映射文件的二级缓存 -->
    <cache />

3.实体类序列化

实体类要实现Serializable

因为实体类需要序列化到硬盘上 以便于缓存

4.栗子

在两个session中

    @Test
    public void cache11() {
        // 获得session
        SqlSession session = sqlSessionFactory.openSession();
        try {
            // 发送SQL语句
            Person person = session.selectOne("com.yutouxiuxiu.Person.selectPersonById", 1);
        } finally {
            session.close();
        }
        
        // 再次获得session
        SqlSession session1 = sqlSessionFactory.openSession();
        try {
            // 发送SQL语句
            Person person1 = session1.selectOne("com.yutouxiuxiu.Person.selectPersonById", 1);
        } finally {
            session1.close();
        }
    }

以上的栗子中只会发出一个SQL语句

5.对每条SQL语句进行缓存控制

默认useCache是为true的 如果设置为false 它就不会对这条SQL语句进行二级缓存

    <select id="selectPersonById" useCache="false" parameterType="java.lang.Integer" resultMap="personRM">
        select * from person where person_id = #{id}
    </select>

6.刷新缓存 不刷新缓存

如果在一个sessionFactory中 一次查询之后 又进行了数据的修改 再次查询 就会重新发出SQL语句

默认是刷新缓存的 也可以在修改的语句上进行设置 不让它自动刷新缓存

<update id="update" flushCache="false" ...

如果在insert语句上加上flushCache="false"会出现什么情况呢

比如说查询一个集合 然后又插入了一条数据 又查询该集合 就不会刷新缓存 查不到新插入的数据

delete的情况 也类似

update是针对单个实体的 insert和delete是针对集合的

7.cache标签的属性

在Mapping文件中

（1）逐出策略 eviction

LRU  -  最近最少使用 对象放到缓存里面了 这个对象很久（可以设置时间）都没有人用 就逐出缓存

FIFO  -  先进先出 队列 按照对象进入缓存的顺序来移除（用的最多）

SOFT  -  软引用 利用垃圾回收机制（不怎么用）

WEAK  -  若引用 利用垃圾回收机制（不怎么用）

（2）定时刷新 flushInterval

ms为单位 每多长时间刷新缓存 定时任务 一直在后台跑 对性能有影响

（3）缓存对象数量 size

缓存中能存放多少对象

（4）缓存制度 readOnly

如果为true 缓存不能人为修改 只能由MyBatis自己操作
